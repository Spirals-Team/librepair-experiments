062/**
063 * This has developed over the last couple of months/years towards an increasingly pluggable module.  The current (dec'2011)
064 * approach consists of the following elements (and presumably more, developed by mzilske):<ul>
065 * <li> QSim itself should have all basic functionality to execute a typical agent plan, i.e. activities and legs.  In this basic
066 * version, all legs are teleported.
067 * <li> In addition, there are "engines" that plug into QSim.  Those are time-step driven, as is QSim.  Many engines move
068 * particles around, i.e. they execute the different modes.  Others are responsible for, e.g., time-variant networks or signals.
069 * <li> A special engine is the netsim engine, which is the original "queue"
070 * engine.  It is invoked by default, and it carries the "NetsimNetwork" for which there is a getter.
071 * <li> Engines that move particles around need to be able to "end legs".
072 * This used to be such that control went to the agents, which
073 * reinserted themselves into QSim.  This has now been changed: The agents compute their next state, but the engines are
074 * responsible for reinsertion into QSim.  For this, they obtain an "internal interface" during engine addition.  Naming
075 * conventions will be adapted to this in the future.
076 * <li> <i>A caveat is that drivers that move around other agents (such as TransitDriver, TaxicabDriver) need to become
077 * "engines".</i>  Possibly, something that executes a leg is not really the same as an "engine", but this is what we have
078 * for the time being.
079 * <li> Engines that offer new modes also need to be registered as "DepartureHandler"s.
080 *  * </ul>
081 * Future plans include: pull the agent counter write methods back into QSim (no big deal, I hope); pull the actstart/end,
082 * agent departure/arrival back into QSim+engines; somewhat separate the teleportation engine and the activities engine from the
083 * framework part of QSim.
084 * <p></p>
085 * @author dstrippgen
086 * @author mrieser
087 * @author dgrether
088 * @author knagel
089 */
090public final class QSim extends Thread implements VisMobsim, Netsim, ActivityEndRescheduler {

215        @Override
216        public void run() {
217                try {
218                        // Teleportation must be last (default) departure handler, so add it
219                        // only before running.
220                        addDepartureHandler(this.teleportationEngine);
221                        prepareSim();
222                        this.listenerManager.fireQueueSimulationInitializedEvent();
223
224                        // Put agents into the handler for their first ("overnight") action,
225                        // probably the ActivityEngine. This is done before the first
226                        // beforeSimStepEvent, because the expectation seems to be
227                        // (e.g. in OTFVis), that agents are doing something
228                        // (can be located somewhere) before you execute a sim step.
229                        // Agents can abort in this loop already, so we iterate over
230                        // a defensive copy of the agent collection.
231                        for (MobsimAgent agent : new ArrayList<>(this.agents.values())) {
232                                arrangeNextAgentAction(agent);
233                        }
234
235                        // do iterations
236                        boolean doContinue = true;
237                        while (doContinue) {
238                                doContinue = doSimStep();
239                        }
240                } finally {
241                        // We really want to perform that. For instance, with QNetsimEngine, threads are cleaned up in this method.
242                        // Without this finally, in case of a crash, threads are not closed, which lead to process hanging forever
243                        // at least on the eth euler cluster (but not on our local machines at ivt!?) td oct 15
244                        cleanupSim();
245                }
246        }

304        void cleanupSim() {
305                this.listenerManager.fireQueueSimulationBeforeCleanupEvent();
306
307                boolean gotException = false;
308                for (MobsimEngine mobsimEngine : mobsimEngines) {
309                        try {
310                                // make sure all engines are cleaned up
311                                mobsimEngine.afterSim();
312                        }
313                        catch (Exception e) {
314                                log.error("got exception while cleaning up", e);
315                        }
316                }
317
318                if (gotException) throw new RuntimeException( "got exception while cleaning up the QSim. Please check the error messages above for details.");
319                events.finishProcessing();
320                if (analyzeRunTimes) {
321                        log.info("qsim internal cpu time (nanos): " + qSimInternalTime);
322                        for (Entry<MobsimEngine, AtomicLong> entry : this.mobsimEngineRunTimes.entrySet()) {
323                                log.info(entry.getKey().getClass().toString() + " cpu time (nanos): " + entry.getValue().get());                                
324                        }
325                        log.info("");
326                        if ( this.netEngine instanceof QNetsimEngine ) {
327                                ((QNetsimEngine)this.netEngine).printEngineRunTimes();
328                                // (yy should somehow be in afterSim()).
329                        }
330                }
331        }

333        /**
334         * Do one step of the simulation run.
335         *
336         * @return true if the simulation needs to continue
337         */
338        /*package*/ boolean doSimStep() {
339                if (analyzeRunTimes) this.startTime = System.nanoTime();
340
341                final double now = this.getSimTimer().getTimeOfDay();
342
343                this.listenerManager.fireQueueSimulationBeforeSimStepEvent(now);
344                
345                if (analyzeRunTimes) this.qSimInternalTime += System.nanoTime() - this.startTime;
346                
347                /*
348                 * The WithinDayEngine has to perform its replannings before
349                 * the other engines simulate the sim step.
350                 */
351                if (this.withindayEngine != null) {
352                        if (analyzeRunTimes) startTime = System.nanoTime();
353                        this.withindayEngine.doSimStep(now);
354                        if (analyzeRunTimes) this.mobsimEngineRunTimes.get(this.withindayEngine).addAndGet(System.nanoTime() - this.startTime);
355                }
356
357                // "added" engines
358                for (MobsimEngine mobsimEngine : this.mobsimEngines) {
359                        if (analyzeRunTimes) this.startTime = System.nanoTime();
360                        
361                        // withindayEngine.doSimStep(time) has already been called
362                        if (mobsimEngine == this.withindayEngine) continue;
363
364                        mobsimEngine.doSimStep(now);
365                        
366                        if (analyzeRunTimes) this.mobsimEngineRunTimes.get(mobsimEngine).addAndGet(System.nanoTime() - this.startTime);
367                }
368
369                if (analyzeRunTimes) this.startTime = System.nanoTime();
370                
371                // console printout:
372                this.printSimLog(now);
373                boolean doContinue =  (this.agentCounter.isLiving() && (this.stopTime > now));
374                this.events.afterSimStep(now);
375                this.listenerManager.fireQueueSimulationAfterSimStepEvent(now);
376
377                final QSimConfigGroup qsimConfigGroup = this.scenario.getConfig().qsim();
378                if ( qsimConfigGroup.getSimEndtimeInterpretation()==EndtimeInterpretation.onlyUseEndtime ) {
379                        if ( now > qsimConfigGroup.getEndTime() ) {
380                                doContinue = false ;
381                        } else {
382                                doContinue = true ;
383                        }
384                }
385
386                if (doContinue) {
387                        this.simTimer.incrementTime();
388                }
389                
390                if (analyzeRunTimes) this.qSimInternalTime += System.nanoTime() - this.startTime;
391
392                return doContinue;
393        }

395        public void insertAgentIntoMobsim(final MobsimAgent agent) {
396                if (this.agents.containsKey(agent.getId())) {
397                        throw new RuntimeException("Agent with same Id (" + agent.getId().toString() + ") already in mobsim; aborting ... ") ;
398                }
399                this.agents.put(agent.getId(), agent);
400                this.agentCounter.incLiving();
401        }

403        private void arrangeNextAgentAction(final MobsimAgent agent) {
404                switch( agent.getState() ) {
405                case ACTIVITY:
406                        arrangeAgentActivity(agent);
407                        break ;
408                case LEG:
409                        this.arrangeAgentDeparture(agent);
410                        break ;
411                case ABORT:
412                        this.events.processEvent( new PersonStuckEvent(this.simTimer.getTimeOfDay(), agent.getId(), agent.getCurrentLinkId(), agent.getMode()));
413
414                        // NOTE: in the same way as one can register departure handler or activity handler, we could allow to
415                        // register abort handlers.  If someone ever comes to this place here and needs this.  kai, nov'17
416                        
417                        this.agents.remove(agent.getId()) ;
418                        this.agentCounter.decLiving();
419                        this.agentCounter.incLost();
420                        break ;
421                default:
422                        throw new RuntimeException("agent with unknown state (possibly null)") ;
423                }
424        }

436         * current activity. The simulation can then put the agent onto its vehicle
437         * on a link or teleport it to its destination.

439         */
440        private void arrangeAgentDeparture(final MobsimAgent agent) {
441                double now = this.getSimTimer().getTimeOfDay();
442                Id<Link> linkId = agent.getCurrentLinkId();
443                Gbl.assertIf( linkId!=null );
444                events.processEvent(new PersonDepartureEvent(now, agent.getId(), linkId, agent.getMode()));
445
446                for (DepartureHandler departureHandler : this.departureHandlers) {
447                        if (departureHandler.handleDeparture(now, agent, linkId)) {
448                                return;
449                        }
450                }
451                log.warn("no departure handler wanted to handle the departure of agent " + agent.getId());
452                // yy my intuition is that this should be followed by setting the agent state to abort. kai, nov'14
453
454        }

460        private void initSimTimer() {
461                QSimConfigGroup qSimConfigGroup = this.scenario.getConfig().qsim();
462                Double configuredStartTime = qSimConfigGroup.getStartTime();
463                this.stopTime = qSimConfigGroup.getEndTime();
464                if (configuredStartTime == Time.UNDEFINED_TIME) {
465                        configuredStartTime = 0.0;
466                }
467                if ((this.stopTime == Time.UNDEFINED_TIME) || (this.stopTime == 0)) {
468                        this.stopTime = Double.MAX_VALUE;
469                }
470
471                double simStartTime;
472                if (QSimConfigGroup.StarttimeInterpretation.maxOfStarttimeAndEarliestActivityEnd.equals(qSimConfigGroup.getSimStarttimeInterpretation())) {
473                        double firstAgentStartTime = calculateFirstAgentStartTime();
474                        simStartTime = Math.floor(Math.max(configuredStartTime, firstAgentStartTime));
475                } else if (QSimConfigGroup.StarttimeInterpretation.onlyUseStarttime.equals(qSimConfigGroup.getSimStarttimeInterpretation())) {
476                        simStartTime = configuredStartTime;
477                } else {
478                        throw new RuntimeException("unkonwn starttimeInterpretation; aborting ...");
479                }
480
481                this.simTimer.setSimStartTime(simStartTime);
482                this.simTimer.setTime(simStartTime);
483
484        }
485
486        private double calculateFirstAgentStartTime() {
487                double firstAgentStartTime = Double.POSITIVE_INFINITY;
488                for (MobsimAgent agent : agents.values()) {
489                        firstAgentStartTime = Math.min(firstAgentStartTime, agent.getActivityEndTime());
490                }
491                return firstAgentStartTime;
492        }

498        private void printSimLog(final double time) {
499                if (time >= this.infoTime) {
500                        //              if(true){
501                        this.infoTime += INFO_PERIOD;
502                        Date endtime = new Date();
503                        long diffreal = (endtime.getTime() - this.realWorldStarttime
504                                        .getTime()) / 1000;
505                        double diffsim = time - this.simTimer.getSimStartTime();
506                        log.info("SIMULATION (NEW QSim) AT " + Time.writeTime(time)
507                                        + " : #Veh=" + this.agentCounter.getLiving() + " lost="
508                                        + this.agentCounter.getLost() + " simT=" + diffsim
509                                        + "s realT=" + (diffreal) + "s; (s/r): "
510                                        + (diffsim / (diffreal + Double.MIN_VALUE)));
511
512                        Gbl.printMemoryUsage();
513                }
514        }

652                // used (and thus implicitly tested) by bdi-abm-integration project.  A separate core test would be good. kai, feb'18
