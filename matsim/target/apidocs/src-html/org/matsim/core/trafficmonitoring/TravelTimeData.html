021package org.matsim.core.trafficmonitoring;
022
023public interface TravelTimeData {
024
025        public abstract void resetTravelTimes();
026        
027        /** Selectively reset the travel time accounting for a certain time slot.  This may be necessary for some version of
028         * "consolidateData", but right now it is not needed.  kai, oct'11
029         */
030//      public abstract void resetTravelTime( final int timeSlot ) ;
031
032        abstract void addTravelTime(final int timeSlot, final double traveltime);
033        
034        /**
035         * A method to set the travel time directly, to handle some special cases.
036         */
037        abstract void setTravelTime( final int timeSlot, final double traveltime ) ;
038
039        /**Returns the travel time (presumably in seconds), given both the time slot and the time-of-day for the request.
040         * <p></p>
041         * Design thoughts:<ul>
042         * <li> yy It does not make sense to me that both "timeSlot" and "now" are needed.  I guess that historically it was
043         * "timeSlot", and then "now" was needed for the time-dependent network.  But I would think that the conversion
044         * from "now" to "timeSlot" should be done inside the data class (implementation hiding) and so the "timeSlot" 
045         * argument should be removed.  kai, may'2011 </li>
046         * <li>Indead, this doesn't make sense at a quick glance. This is an optimization that saves lots of memory. For each
047         * link of the network a TravelTimeData Object is created. Each of these objects would need one (Map implementation) 
048         * or two (Array implementation) additional fields that hold the information that is required to calculate the time slot
049         * from the time of day. dg, april'2013. </li>
050         * <li> Well, one could insert the algo to compute the index from the time-of-day into the TravelTimeData implementing class. kai, jan'16
051         * <li> The main reason by both are needed seems to be the getFreeSpeed(...) at some point, which needs "now" as an argument.  
052         * We could, alternatively, return some "undefined" travel time (or an infinite standard deviation), and then now somewhere else
053         * that we need to use free speed as the alternative.  kai, jan'16
054         * </ul> 
055         */
056        abstract double getTravelTime(final int timeSlot, final double now);
057
058}
