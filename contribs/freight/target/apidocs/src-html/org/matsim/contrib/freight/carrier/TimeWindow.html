007/**
008 * Q: What happens/should happen if the time window is not sufficient to unload, or
009 * the vehicle arrives after the time window?
010 * <p></p>
011 * A: First, I (=sschroeder) interpret time-window as period of time where an
012 * (un)loading operation can start (which does not mean that it has to be
013 * finished by the end of the time window). This is - I guess - in line with
014 * most OR literature dealing with time-windows. There are 3 options to deal
015 * with late arrivals:<ol> 
016 * <li> agent does not (un)load and immediately goes to the
017 * next activity. missed operation is then penalyzed in the scoring part (which
018 * is - I think - far from reality) 
019 * <li> agent (un)loads at arr_time and late
020 * arrTime is penalyzed in the scoring part 
021 * <li> agent gets an extra waiting_time
022 * before (un)loading. optionally, late arr_time is then penalyzed in the
023 * scoring part additionally.
024 * </ol>
025 * I guess that it depends on the freight sector. In last mile of the CEP market
026 * (parcels), I'd prefer option 2. When it comes to long haul transport, e.g.
027 * delivering Bosch with (raw) material, option 3 might be better. Here, I know
028 * that carriers need to book time-windows. If they are in time, they can unload
029 * immediately. If not, they need to wait for the next available time slot which
030 * might be 3hours later. I think both options (2,3) might be (easily)
031 * incorporated with WithinDayReplanning(ReScheduling)-Module.
032 * <p></p>
033 * 
034 * @author (of code) sschroeder (of docu) kai based on sschroeder
035 */
036
037public class TimeWindow {
038
039        public static TimeWindow newInstance(double start, double end) {
040                return new TimeWindow(start, end);
041        }
042
043        private final double start;
044
045        private final double end;
046
047        private TimeWindow(final double start, final double end) {
048                this.start = start;
049                this.end = end;
050        }
051
052        public double getStart() {
053                return start;
054        }
055
056        public double getEnd() {
057                return end;
058        }
059
060        @Override
061        public String toString() {
062                return "[start=" + Time.writeTime(start) + ", end=" + Time.writeTime(end) + "]";
063        }
064
065}
